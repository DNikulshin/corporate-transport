# Quick Start - Мобильное приложение

## 🚀 Быстрый старт

### 1. Установка зависимостей (если еще не установлены)

```bash
npm install
```

### 2. Настройка окружения

```bash
cp .env.example .env
```

Откройте `.env` и убедитесь что URL правильные:

- Для iOS симулятора: `http://localhost:4000`
- Для Android эмулятора: `http://10.0.2.2:4000`

### 3. Запустите бэкенд (в отдельном терминале)

```bash
cd ../../backend
npm run dev
```

Убедитесь что бэкенд запущен на порту 4000:

```
🚀 Backend running on http://localhost:4000
```

### 4. Запустите мобильное приложение

```bash
cd ../mobile
npm start
```

### 5. Выберите устройство

После запуска вы увидите:

```
› Press │a│ to open Android.
› Press │i│ to open iOS simulator.
› Press │w│ to open web browser.
› Press │r│ to reload the app.
```

#### Вариант A: Android эмулятор

1. Нажмите `a` в терминале
2. Дождитесь загрузки эмулятора
3. Приложение должно открыться автоматически

#### Вариант B: iOS симулятор (только macOS)

1. Нажмите `i` в терминале
2. Дождитесь запуска Xcode Simulator
3. Приложение откроется автоматически

#### Вариант C: Реальное устройство

1. Установите приложение **Expo Go** на телефон:
   - iOS: App Store
   - Android: Google Play / F-Droid
2. В приложении Expo Go отсканируйте QR код из терминала

3. Приложение загрузится на устройстве

### Экран авторизации

При первом запуске вы увидите экран входа:

```

│  Корпоративный транспорт    │

│  Пароль: [__________]       │
│                             │
│      [ Войти ]              │

```

```
Пароль: password123
```

После входа откроется экран с картой и кнопкой **"Начать смену"**.

```

```

После входа откроется только карта с маркерами автомобилей.

## 🔧 Отладка

### Debug меню

В экспресс режиме:

- Встряхните устройство (или нажмите Cmd+D на симуляторе)
- Откроется меню разработчика

### Логи

Все логи выводятся в терминал где запущено `npm start`

Для просмотра логов React Native:

```bash
npx react-native log-android  # Android
npx react-native log-ios      # iOS
```

### Проверка соединения

Если приложение не подключается:

1. Проверьте что бэкенд запущен:

```bash
curl http://localhost:4000/health
```

2. Проверьте CORS в консоли бэкенда

3. Для Android: убедитесь что используете `10.0.2.2` вместо `localhost`

4. Проверьте файрвол (порт 4000)

### Hot Reload

Для быстрой перезагрузки приложения:

- Нажмите `r` в терминале
- Или встряхните устройство → "Reload"

## ⚙️ Настройки

### Изменить интервал GPS

Откройте `app/config/env.ts`:

```typescript
gpsIntervalMs: 2000, // каждые 2 секунды
```

### Изменить порт

Если бэкенд на другом порту, обновите `.env`:

```env
API_URL=http://localhost:3001
WS_URL=ws://localhost:3001
```

### Production сборка

```bash
eas build --platform android
eas build --platform ios
```

Требуется настроить EAS аккаунт.

## 🐛 Частые проблемы

### "Network request failed"

- Проверьте что бэкенд запущен
- Для Android используйте `10.0.2.2`
- Проверьте файрвол

### "Location permission denied"

- iOS: Проверьте настройки приватности
- Android: Дайте разрешение в настройках эмулятора

### Карта не загружается

- Проверьте интернет соединение
- Для реального устройства: включите GPS

### WebSocket не подключается

- Проверьте консоль бэкенда
- Убедитесь что JWT токен действителен
- Проверьте Redis (должен быть запущен)
